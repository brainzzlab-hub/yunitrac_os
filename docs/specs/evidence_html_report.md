# Evidence HTML Report (Deterministic, No PII)

Scope
- Produce a single-file HTML summary for non-dev stakeholders.
- Deterministic: no timestamps, no env-dependent data, stable ordering.
- No PII: only hashes/sizes/labels/counts/SecCodes; never include task text or model outputs.
- Outside boundary; does not modify runtime crates.

Required input
- `artifacts/audit/evidence_pack_index.json`

Optional inputs (included if present)
- `artifacts/control_plane/runbook_report.json`
- `artifacts/security/gdpr_findings.json`
- `artifacts/sbom/license_policy_report.json`
- `artifacts/audit/index_manifest.json`
- `artifacts/incident/index_manifest.json`

Outputs
- `artifacts/compliance/evidence_report.html` (deterministic HTML)
- `artifacts/compliance/evidence_report.json` (label+count only; sorted keys; newline)

Sections (fixed order)
1) Header + disclaimer (“Evidence summary, not legal advice”)
2) Overall status (PASS/FAIL from runbook_report if available; else UNKNOWN)
3) Evidence inventory (name/path/sha256/size from evidence_pack_index, sorted by name)
4) GDPR summary (label+count only, else MISSING)
5) SBOM/license summary (label+count only, else MISSING)
6) Manifests present flags (audit/incident yes/no)
7) Footer “Generated by yunitrack_os | Stateless | No data retained”

Failure handling (bounded)
- stdout: `PASS: evidence_html_report` or `FAIL: evidence_html_report <SecCode>`
- SecCodes: EHTML_MISSING_REQUIRED, EHTML_BAD_JSON, EHTML_IO_FAIL
