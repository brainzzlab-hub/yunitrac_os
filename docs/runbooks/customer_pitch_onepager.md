# yunitrac_os v0.3 dual-profile — Customer Pitch (One Page)

## Profiles at a glance
- **Enterprise (E):** diode-first, production-grade, no MVP shortcuts. Secure runner with no time/net/fs/rand/threads/stdout; one-way diode; PKCS#11 approvals; audit chain; anti-replay.
- **Lite (L):** scoped determinism for local/CI parity; soft-diode assumption; bounded SecCode result only; equality proven on declared REQUIRED_EQUALITY_SET.

## Proofable claims (mechanically enforced under stated assumptions)
- Determinism: B==C for declared equality set (E: outputs/audit/metrics/security/hashes; L: scope_manifest set).
- Audit chain: HMAC chain verified (fail-closed) in enterprise proof.
- Approvals: unsigned rejected; PKCS#11-signed accepted; nonce==0 rejected; duplicates rejected; cross-run ledger in ingress.
- Buckets: AUDIT/SECURITY content-free; METRICS numeric-only, bounded, quantized.
- Diode-first: secure_runner writer-only; ro_out_receiver reader-only; gate_no_ack enforces no backchannel.

## Non-claims / out of scope
- No performance/SLA guarantees.
- No host OS hardening beyond documented sandbox wrapper (deployers must harden host).
- Lite profile does not claim PKCS#11 chain verification or hardware diode.
- No data-at-rest encryption commitments (deployment choice).

## Security posture checklist (customer-facing)
- One-way diode (E) / soft-diode (L), no feedback to ac beyond bounded SecCode.
- Deterministic outputs within declared scope.
- Audit chain verified; fail-closed.
- Approvals via PKCS#11; no private keys in repo.
- Anti-replay: nonce rules + ledger (E).
- GDPR-minimal buckets: no payload content in AUDIT/SECURITY; METRICS numeric-bounded/quantized.

## Artifacts & paths (stable)
- Enterprise: `artifacts/enterprise/runA|B|C/`, `artifacts/enterprise/evidence_bundle/`, `artifacts/proof_report_enterprise.json`.
- Lite: `artifacts/lite/` (scope_manifest, evidence_bundle, tmp_B/tmp_C), `artifacts/proof_report_lite.json`.

## Decision aids (platform comparison, same build)
- Compare equality-set hashes (B vs C) only on declared REQUIRED_EQUALITY_SET.
- Verify diode/no-feedback gates in scripts (gate_no_ack, gate_boundary_deps).
- Check bucket policy JSONs and manifest hashes; canonical JSON (sorted keys).
- Confirm no timestamps or environment data in equality-scoped artifacts.
